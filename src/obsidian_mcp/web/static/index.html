          Dataview — Query Language Reference
          
        
          DATAVIEW PLUGIN  (github.com/blacksmithgu/obsidian-dataview)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DATA SOURCES
  Frontmatter YAML (between --- fences at top of note)
  Inline fields:  key:: value  or  [key:: value]  or  (key:: hidden)

QUERY TYPES
  TABLE field1, field2 FROM "folder" WHERE condition SORT field DESC LIMIT n
  LIST   [field]       FROM source   WHERE condition SORT field
  TASK                 FROM source   WHERE !completed GROUP BY file.folder
  CALENDAR date        FROM source

COMMON IMPLICIT FIELDS (always available)
  file.name      file.path    file.size    file.ctime   file.mtime
  file.tags      file.etags   file.inlinks file.outlinks file.aliases
  file.starred   file.day     file.tasks   file.lists

FROM SOURCES
  FROM "folder"          — notes inside folder (recursive)
  FROM #tag              — notes with tag
  FROM [[Note]]          — notes linking to Note
  FROM outgoing([[Note]])— notes Note links to
  FROM "folder" OR #tag  — combine sources

WHERE EXPRESSIONS
  status = "active"      status != "done"
  priority >= 2          due < date(today)
  contains(tags, "#project")
  !completed             length(file.tasks) > 0
  date(file.ctime) > date("2026-01-01")

SORT / LIMIT
  SORT file.mtime DESC   SORT priority ASC, due ASC
  LIMIT 20

FLATTEN / GROUP BY
  FLATTEN list-field AS item
  GROUP BY file.folder

EXAMPLES
  ┌─ Active projects ──────────────────────────────────────────
  │ ```dataview
  │ TABLE status, priority, due
  │ FROM "Projects"
  │ WHERE status != "done"
  │ SORT priority DESC
  │ ```
  ├─ Open tasks due this week ──────────────────────────────────
  │ ```dataview
  │ TASK
  │ FROM ""
  │ WHERE !completed AND due <= date(today) + dur(7 days)
  │ SORT due ASC
  │ ```
  └─ Recently modified ─────────────────────────────────────────
    ```dataview
    TABLE file.mtime AS Modified
    FROM ""
    SORT file.mtime DESC
    LIMIT 10
    ```

AI USAGE NOTES
  • MCP generates static dataview blocks — the plugin renders them live in Obsidian.
  • Always add inline fields (key:: value) when you want a note to be queryable.
  • Use file.mtime for recency queries, not created date.
  • Prefer FROM "folder" over FROM "" for performance in large vaults.
          
        
          
      
          

      
      
          
        
          Tasks Plugin — Emoji Syntax Reference
          
        
          TASKS PLUGIN  (github.com/obsidian-tasks-group/obsidian-tasks)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TASK FORMAT
  - [ ] Task description  [emoji fields]  [#tags]  [^block-ref]

TASK STATES
  - [ ]  open          - [x]  done
  - [-]  cancelled     - [/]  in-progress

EMOJI FIELDS (must appear AFTER task text, parsed right-to-left)
  📅 YYYY-MM-DD   due date
  ⏳ YYYY-MM-DD   scheduled date
  🛫 YYYY-MM-DD   start date
  ✅ YYYY-MM-DD   done date (auto-added on completion)
  ❌ YYYY-MM-DD   cancelled date
  🔁 rule          recurrence  (e.g. "every week", "every month on the 1st")

PRIORITY EMOJIS
  ⏫  highest       🔼  high
  🔽  low           ⏬  lowest
  (no emoji = normal priority)

RECURRENCE RULES (examples)
  every day           every week
  every month         every year
  every weekday       every 2 weeks
  every month on the 15th

ORDERING RULE (important for AI)
  Tasks reads BACKWARDS from line end. Place emojis in this order:
  description  🔁 recurrence  🛫 start  ⏳ scheduled  📅 due  priority  #tags  ^block

EXAMPLES
  - [ ] Write daily standup 🔁 every weekday 📅 2026-06-14 🔼
  - [ ] Review Proxmox backups 🔁 every month 📅 2026-07-01
  - [x] Setup vault structure ✅ 2026-06-10
  - [ ] Read AI paper 📅 2026-06-20 ⏫

QUERY BLOCK (inside a note)
  ```tasks
  not done
  due before 2026-07-01
  priority is high
  sort by due
  ```

AI USAGE NOTES
  • Always put emoji fields AFTER the task text — Tasks parses right-to-left.
  • Anything non-emoji after the description breaks date/priority parsing.
  • Use 🔁 recurrence before date fields on the same line.
  • The MCP create_task tool handles formatting automatically.
  • Tags (#tag) and block refs (^id) go at the very end.
          
        
          
      
          

      
      
          
        
          Templater — Syntax & Functions Reference
          
        
          TEMPLATER PLUGIN  (github.com/SilentVoid13/Templater)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

COMMAND TAGS
  <% expression %>    — insert result of expression
  <%* js code %>      — execute JavaScript (use tR += "..." to append)
  <%+ expression %>   — dynamic command (only re-evaluates in preview)
  <%- expression %>   — whitespace-trimming variant

tp OBJECT — BUILT-IN MODULES

  tp.date
    tp.date.now()                        → "2026-06-13"  (ISO)
    tp.date.now("YYYY-MM-DD")            → "2026-06-13"
    tp.date.now("dddd, MMMM Do YYYY")    → "Saturday, June 13th 2026"
    tp.date.now("YYYY-MM-DD", -1)        → yesterday
    tp.date.now("YYYY-MM-DD", 7)         → 7 days from now
    tp.date.tomorrow("YYYY-MM-DD")       → tomorrow
    tp.date.yesterday("YYYY-MM-DD")      → yesterday
    tp.date.weekday("YYYY-MM-DD", 0)     → this Monday

  tp.file
    tp.file.title          → current file name (without .md)
    tp.file.folder(true)   → full folder path
    tp.file.path(true)     → full file path
    tp.file.creation_date("YYYY-MM-DD")
    tp.file.last_modified_date("YYYY-MM-DD")
    tp.file.cursor()       → place cursor here after render
    tp.file.cursor(1)      → numbered cursor jump point
    tp.file.tags           → array of file tags

  tp.system
    await tp.system.prompt("Enter title:")         → user input string
    await tp.system.suggester(["A","B"], ["a","b"])→ user picks from list
    await tp.system.clipboard()                    → clipboard content

  tp.frontmatter
    tp.frontmatter.status   → reads frontmatter field "status"

MOMENT.JS DATE FORMATS
  YYYY → 2026    MM → 06    DD → 13
  dddd → Saturday    ddd → Sat    Do → 13th
  HH:mm → 14:35

STATIC vs DYNAMIC (MCP behaviour)
  MCP resolves these at generation time (no Obsidian required):
    tp.date.now()   tp.date.now("YYYY-MM-DD")   tp.file.title
  MCP preserves these as-is for Templater to resolve when opened:
    <%* js blocks %>   tp.system.prompt()   tp.file.cursor()

TEMPLATE TYPES SUPPORTED BY MCP
  concept   project   journal   meeting   reference   system

EXAMPLES
  ---
  title: <% tp.file.title %>
  created: <% tp.date.now("YYYY-MM-DD") %>
  type: project
  status: active
  ---
  # <% tp.file.title %>
  ## Tasks
  - [ ] First task 📅 <% tp.date.now("YYYY-MM-DD", 7) %>

AI USAGE NOTES
  • MCP resolves static tp.* calls at note-generation time.
  • Dynamic <%* %> blocks are preserved as-is for Templater.
  • Use apply_template tool to render a template into a new note.
  • Templates folder defaults to "Templates/" — configure via env.
          
        
          
      
          

      
      
          
        
          Excalidraw — File Format & AI Generation Guide
          
        
          EXCALIDRAW PLUGIN  (github.com/zsviczian/obsidian-excalidraw-plugin)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

FILE FORMAT  (.excalidraw.md)
  A markdown file with special frontmatter + embedded JSON block.

  ---
  excalidraw-plugin: parsed
  tags: [excalidraw, diagram]
  ---
  # Diagram Title

  ```json
  {
    "type": "excalidraw",
    "version": 2,
    "elements": [ ... ],
    "appState": { "viewBackgroundColor": "transparent" }
  }
  ```

ELEMENT TYPES
  rectangle   ellipse   diamond   arrow   line   text   freedraw
  image       frame     embeddable

COMMON ELEMENT FIELDS
  {
    "id": "el-0001",
    "type": "rectangle",
    "x": 100, "y": 200,
    "width": 180, "height": 60,
    "strokeColor": "#1e1e1e",
    "backgroundColor": "#dbe9ff",
    "fillStyle": "solid",       // hachure | solid | cross-hatch | dots
    "strokeWidth": 2,
    "strokeStyle": "solid",     // solid | dashed | dotted
    "roughness": 1,             // 0=polished, 1=hand-drawn, 2=rough
    "opacity": 100,
    "text": "My Node",
    "fontSize": 16,
    "fontFamily": 3,            // 1=hand, 2=normal, 3=mono
    "textAlign": "center",
    "angle": 0
  }

ARROW ELEMENT (additional fields)
  "points": [[0,0],[200,0]]    // relative coordinates from x,y
  "startBinding": null         // or { "elementId": "el-0001" }
  "endBinding":   null

NODE STYLES USED BY MCP
  service  → rectangle, #dbe9ff (blue)
  database → ellipse,   #fce8ff (purple)
  queue    → rectangle, #fff3bf (yellow)
  external → rectangle, #e8f5e9 (green)
  user     → ellipse,   #ffeedd (orange)

LAYOUTS
  layered  → left-to-right horizontal flow
  grid     → sqrt(n) columns
  radial   → circular around centre

MCP TOOLS
  generate_excalidraw_architecture(path, title, nodes, edges, layout)
  generate_excalidraw_concept_map(path, central_concept, branches)
  excalidraw_parse_elements(path)
  excalidraw_add_annotation(path, text, x, y)

AI USAGE NOTES
  • Always use .excalidraw.md extension for Obsidian Excalidraw plugin to open it.
  • Enable "Decompress Excalidraw JSON" in plugin settings to edit JSON directly.
  • IDs must be unique strings within the file.
  • Arrow points are RELATIVE to the arrow's x,y origin, not absolute.
  • roughness=0 for clean diagrams; roughness=1 for hand-drawn aesthetic.
  • Since Excalidraw 1.6.13+, JSON may be compressed — MCP always writes decompressed.
          
        
          
      
          

      
      
          
        
          Omnisearch — Indexing & Optimisation Guide
          
        
          OMNISEARCH PLUGIN  (github.com/scambier/obsidian-omnisearch)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WHAT OMNISEARCH INDEXES
  • Note file names and paths
  • Frontmatter: title, aliases, tags, all custom fields
  • Full note body content
  • Headings (weighted higher)
  • PDF content (if PDF indexing enabled)

WHAT MAKES A NOTE DISCOVERABLE
  ✓ Descriptive filename (spaces > underscores > slugs)
  ✓ title: in frontmatter (overrides filename in results)
  ✓ aliases: list in frontmatter (alternative names)
  ✓ tags: specific and hierarchical (#home-lab/proxmox)
  ✓ Key terms repeated naturally in body
  ✓ Headings that match common search queries

ALIASES FRONTMATTER
  aliases:
    - Proxmox Hypervisor
    - PVE
    - Proxmox VE

  These let Omnisearch (and Obsidian quick-switcher) find the note
  by any of these names. The MCP add_aliases tool writes this for you.

TAGS — HIERARCHICAL FORMAT
  tags:
    - home-lab
    - home-lab/networking
    - status/active
    - type/project

COMMON SEARCH PATTERNS
  Omnisearch supports:
    exact phrase    "proxmox cluster"
    word prefix     proxy*
    file path       path:Projects/
    tag filter      tag:home-lab
    fuzzy match     (default for single words)

MCP TOOLS
  omnisearch_suggest_aliases(path, limit)   → alias candidates
  omnisearch_add_aliases(path, aliases)     → write to frontmatter
  omnisearch_suggest_keywords(path, limit)  → keyword gaps
  omnisearch_find_poorly_indexed(limit)     → vault audit
  omnisearch_optimise_note(path)            → full report

AI USAGE NOTES
  • Run omnisearch_find_poorly_indexed after bulk note creation.
  • Always set title in frontmatter — it weights higher than filename.
  • Acronyms make great aliases: "Home AI System" → alias "HAS".
  • Tags should be hierarchical: #system/networking not just #networking.
  • Omnisearch indexes in real-time — changes appear immediately.
